# AGENTS.md — AI 에이전트 운영 매뉴얼

이 레포는 DAVIAN Lab의 학회 출장 경험을 누적하는 위키다. 사람과 AI가 같은 규칙으로 갱신할 수 있도록 형식을 고정해 두었다. 이 파일은 30초 안에 운영 규칙을 흡수할 수 있게 만들었으니, 글을 쓰기 전에 한 번은 끝까지 읽기를 권한다.

## 1. 디렉터리 트리

```
.
├── README.md                레포 소개
├── AGENTS.md                이 파일
├── CLAUDE.md                Claude Code 자동 로드용 포인터
├── CONTRIBUTING.md          사람용 기여 가이드 (같은 규칙, 부드러운 톤)
├── general.md               국가/도시 무관 공통 팁
├── countries/
│   ├── README.md            국가 인덱스 (가나다순)
│   └── <country>/
│       ├── README.md        그 나라 공통 정보 (통화/콘센트/시차/비자/통신·결제 일반)
│       └── <city>.md        도시별 정보 (공항/숙소/교통/식당/...)
└── templates/
    ├── country-template.md
    └── city-template.md
```

학회별 별도 문서는 만들지 않는다. 학회 정보는 도시 문서의 각 항목 꼬리표로.

## 2. 시나리오별 절차

### 2.1 새 국가 추가

1. `countries/<country>/` 디렉터리 생성 (kebab-case 영문, 예: `usa`, `japan`, `united-kingdom`).
2. `templates/country-template.md`를 `countries/<country>/README.md`로 복사.
3. 국가 공통 정보를 채운다 (퀵 체크리스트 / 입국 / 일반 문화 / 통신 / 화폐·결제·환전 / 응급 연락처).
4. `countries/README.md`의 가나다순 목록에 한 줄 추가:

```diff
 ## 국가 목록 (가나다순)
+- [독일 (Germany)](germany/README.md)
 - [미국 (USA)](usa/README.md)
```

### 2.2 새 도시 추가

1. `templates/city-template.md`를 `countries/<country>/<city>.md`로 복사 (kebab-case 영문, 예: `new-york`, `san-francisco`).
2. 첫 줄 placeholder 치환:

   ```
   > 이 도시는 [<국가>](README.md) 공통 가이드를 따릅니다. 아래는 <도시명>에 한정된 정보입니다.
   ```

3. 도시 한정 퀵 체크리스트를 채운다 (**국가 공통과 다르거나 도시 단위로만 의미 있는 행만**).
4. 해당 국가 `README.md`의 도시 목록에 한 줄 추가:

```diff
 ## 6. 도시 목록
 - [뉴욕 (New York)](new-york.md) — 미 동부, JFK·LGA·EWR 3공항
+- [샌프란시스코 (San Francisco)](san-francisco.md) — 미 서부, SFO
```

### 2.3 퀵 체크리스트 갱신

본문 표의 행을 새 상태/날짜로 **in-place 교체**. 교체 직전 옛 행을 그대로 잘라 같은 문서의 `## 10. 변경 이력 > ### 퀵 체크리스트 — 옛 상태`(국가 문서는 `## 7. 변경 이력 > ### 퀵 체크리스트 — 옛 상태`)로 이동.

```diff
 ## 0. 퀵 체크리스트 (도시 한정)
 | 항목 | 상태 | 최근 확인 |
 | --- | --- | --- |
-| 트래블월렛 교통카드 (OMNY) | ❌ 미지원 | 김OO (2024.12) |
+| 트래블월렛 교통카드 (OMNY) | ✅ 사용 가능 | 박OO (2026.03) |

 ## 10. 변경 이력 (Archive)
 ### 퀵 체크리스트 — 옛 상태
+| 트래블월렛 교통카드 (OMNY) | ❌ 미지원 | 김OO (2024.12) |
```

### 2.4 호텔·식당·관광지 후기 추가

장소는 H3 헤더 한 개로 모은다. 헤더 형식:

```
### [한국어명 (English)](https://maps.app.goo.gl/...)
```

본문 H3 아래에는 **최신 후기 1~2개만** 둔다. 후기가 3개를 넘기면 가장 오래된 것을 같은 이름의 H4(`#### [한국어명 (English)](...)`)와 함께 `## 10. 변경 이력 > ### 숙소 — 옛 후기`(또는 `### 음식 — 옛 후기`)로 이동. 꼬리표는 이동 시에도 떼지 말 것.

```diff
 ## 3. 숙소
 ### [예시 호텔 A (Example Hotel A)](https://maps.app.goo.gl/example-a)
+- 침구 깨끗하고 학회장까지 도보 10분. — 박OO (CVPR 2026.06)
 - 위치 좋고 조식 보통. — 박OO (CVPR 2025.06)
-- 룸 좁지만 가성비 괜찮음. — 이OO (NeurIPS 2024.12)

 ## 10. 변경 이력 (Archive)
 ### 숙소 — 옛 후기
+#### [예시 호텔 A (Example Hotel A)](https://maps.app.goo.gl/example-a)
+- 룸 좁지만 가성비 괜찮음. — 이OO (NeurIPS 2024.12)
```

### 2.5 폐업·철거 처리

H3 통째로(헤더 + 모든 후기)를 `## 10. 변경 이력 > ### 폐업/철거된 장소`로 이동. 헤더 뒤에 폐업 시점을 한 줄로 메모한다.

```diff
 ## 5. 음식
-### [예시 식당 B (Example Restaurant B)](https://maps.app.goo.gl/example-b)
-- 한식 그리울 때 무난. — 이OO (CVPR 2025.06)

 ## 10. 변경 이력 (Archive)
 ### 폐업/철거된 장소
+#### [예시 식당 B (Example Restaurant B)](https://maps.app.goo.gl/example-b)
+_2026.04 폐업 확인 — 박OO_
+- 한식 그리울 때 무난. — 이OO (CVPR 2025.06)
```

## 3. 절대 하지 말 것 (DO NOT)

- ❌ 기존 행/후기를 **삭제**하지 말 것 — Archive로 이동만.
- ❌ 평가 꼬리표 `— 이름 (학회명 YYYY.MM)`를 떼지 말 것. Archive로 옮겨도 유지.
- ❌ 고유명사를 한국어로만 번역하지 말 것 — `[한국어명 (English)]` 원문 병기.
- ❌ 호텔·식당·관광지 H3에 구글맵 링크를 빠뜨리지 말 것.
- ❌ 퀵 체크리스트의 컬럼 수를 바꾸지 말 것 (정확히 3개: `항목 / 상태 / 최근 확인`).
- ❌ 섹션 헤더 번호를 임의로 바꾸지 말 것. 도시 문서는 `0~10`, 국가 문서는 `0~7` 고정.
- ❌ 명백한 오타/사실 오류 외에는 archive 거치지 않고 본문을 직접 수정하지 말 것.
- ❌ `general.md`에 특정 국가·도시 디테일을 넣지 말 것 — 그건 `countries/`로.

## 4. 필수 형식

| 위치 | 형식 |
|---|---|
| 평가 bullet 꼬리표 (학회 후기) | `— 이름 (학회명 YYYY.MM)` |
| 일반 인프라·팁 꼬리표 (학회 무관) | `— 이름 (YYYY.MM)` |
| 체크리스트 "최근 확인" | `이름 (YYYY.MM)` 또는 `—` (영구 사실: 통화·전압 등) |
| 호텔·식당 H3 | `### [한국어명 (English)](https://maps.google.com/...)` |
| 도시 문서 첫 줄 | `> 이 도시는 [<국가>](README.md) 공통 가이드를 따릅니다. 아래는 <도시명>에 한정된 정보입니다.` |
| 도시 Archive 헤더 | `## 10. 변경 이력 (Archive)` |
| 국가 Archive 헤더 | `## 7. 변경 이력 (Archive)` |

## 5. 도시-국가 분업 규칙

- **모든 도시에서 동일한 사실 → 국가 README에만**. (예: 통화, 콘센트, 비자, 팁 일반론)
- **도시별로 갈리는 사실 → 도시 문서로**. (예: 교통카드, 우버 가능 여부, 한식 접근성)
- **충돌이 생기면 국가 문서를 갱신**하고 도시 문서에는 적지 않는다.
- 도시 퀵 체크리스트에는 **도시 단위로만 의미 있는 항목**만 둔다.

### 자국 도시의 변종 형식

한국 도시는 거주자에게 자명한 정보(시차·치안·현지 교통카드·우버/카카오T·한식 접근성·일반 개요·일반 주의사항)를 적지 않는다. 두 패턴 중 선택:

1. **출국 가이드 변종 (거주지 근처 도시 — 예: 서울)** — `city-template.md` 전체를 따르지 않고 `## 2. 공항` + `## 9. 기여자` + `## 10. 변경 이력`만 유지. 도시 단위 일반 가이드(개요·숙소·교통·음식·기념품·관광·주의사항)는 모두 생략.
2. **부분 슬림 변종 (관광지·출장지 — 예: 경주)** — `city-template.md`를 따르되 자명 행(시차·치안·현지 카드·우버·한식)·자명 섹션(`## 1. 개요`, `## 8. 주의사항`)만 제거. 숙소·교통·음식·기념품·관광 섹션은 외지에서 온 출장자에게 의미 있으므로 유지.

새 자국 도시 추가 시 거주지 권역이면 (1), 관광·출장지면 (2). 두 변종 모두 섹션 번호는 그대로 유지(예: ## 0 다음에 ## 2로 점프해도 OK).
